
## 🎯 Project Overview

This project focuses on:

- **SOLID Principles**: Five design principles that help make software more maintainable, flexible, and scalable
- **Design Patterns**: Reusable solutions to common programming problems
- **C# Implementation**: Real-world examples written in C#

## 📚 SOLID Principles

### 1. Single Responsibility Principle (SRP)
A class should have only one reason to change. It should have only one responsibility.

**Directory**: `SolidPrinciple/SingleResponsibility/`

### 2. Open/Closed Principle (OCP)
Software entities should be open for extension but closed for modification.

**Directory**: `SolidPrinciple/OpenClosed/`

### 3. Liskov Substitution Principle (LSP)
Derived classes must be substitutable for their base classes without breaking the application.

**Directory**: `SolidPrinciple/LiskovSubstitution/`

### 4. Interface Segregation Principle (ISP)
Clients should not be forced to depend on interfaces they don't use. Create smaller, more specific interfaces.

**Directory**: `SolidPrinciple/InterfaceSegregation/`

### 5. Dependency Inversion Principle (DIP)
High-level modules should not depend on low-level modules. Both should depend on abstractions.

**Directory**: `SolidPrinciple/DependencyInversion/`
